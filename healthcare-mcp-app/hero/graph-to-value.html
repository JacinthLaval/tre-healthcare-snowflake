💡 The Problem in Plain English

Imagine 3,200 patients, each with genetic data that affects how their body processes common medications like blood thinners, antidepressants, and heart drugs. Some patients metabolize these drugs too fast (the drug doesn't work), some too slow (the drug builds up to toxic levels). The pattern isn't random — patients naturally cluster into communities with shared genetic profiles.

The question is simple: who needs a different prescription, and how urgently?

The challenge is that finding those communities requires analyzing 3.2 million connections between patients — something a human can't do by hand, and something a traditional database wasn't designed for.

The key insight: An ontology stack builder builds a map. What we needed was a GPS. Not "here are 100 views describing relationships" — but "here's the patient who needs attention right now, here's why, and here's what to do about it."

🎯 Three Questions That Matter

We stripped away complexity and asked three questions that a clinician actually cares about:

👥

"Who else is like this patient?"

Find patients with similar genetic profiles using cosine similarity across 12 pharmacogenomic variants and 6 key genes

🌐

"What's the risk profile of this group?"

Identify natural communities using Louvain clustering — 19 communities emerged with distinct risk patterns

⚠

"What should we do differently?"

Generate specific, actionable alerts: "This patient is a Poor Metabolizer on clopidogrel — switch to prasugrel"

🚀 The Five-Phase Journey

This wasn't built over months by a big team. It was built in a single afternoon by one person paired with Cortex Code (CoCo) — Snowflake's AI coding assistant — following five clear phases:

Phase 1: Materialize the Graph

Ran NVIDIA cuGraph on a GPU to compute patient-to-patient similarity (3.2M edges) and community detection (Louvain clustering → 19 communities, 3,202 patients). Wrote results to permanent Snowflake tables. Shut down the GPU.

Phase 2: Create Risk Alerts

Built a SQL view that joins communities, membership, and clinical data to produce 1,977 pharmacogenomic alerts with priority levels (214 HIGH, 1,138 MODERATE, 625 STANDARD) and specific recommended actions.

Phase 3: Teach the AI Agent

Expanded the Cortex Analyst semantic view from 169 to 279 description rows, adding all graph tables. Updated the AI agent to understand community membership, similarity scores, and risk tiers.

Phase 4: Make It Self-Refreshing

Converted community tables to Dynamic Tables with 1-day lag — when new patients are added, the alerts update automatically. Suspended the GPU compute pool to $0/hr.

Phase 5: Put It in the Clinician's Hands

Added a Risk Alerts tab to the Healthcare MCP app (React Native / Expo). Summary cards, priority filters, search, a phenotype-by-ancestry heatmap, and expandable patient detail cards — all powered by direct SQL queries against the materialized graph tables. No GPU needed at query time.
